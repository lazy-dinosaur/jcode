## 5. 구현 디자인 (요약)

### M19 (config hot-reload)

- `OnceLock<Config>` → `OnceLock<Mutex<ConfigCache>>`
- 매 `config()` 호출에서 `stat()` → mtime 비교 (500ms debounce)
- 변경 시 `Config::try_load()` → 성공 시 `Box::leak` 으로 새 `&'static`
- 실패 시 last-good 유지 (warn 만 로깅, 캐시 poison 안 됨)
- public `&'static Config` signature 100% 보존 (callers 무수정)
- 신규 API: `force_reload_config()`, `reset_config_cache_for_tests()` (cfg(test))
- 신규: `Config::try_load() -> anyhow::Result<Self>`
- `set_default_model` 의 옛 OnceLock 우회 워크어라운드 제거
  (`force_reload_config()` 한 줄로 대체)

### M20 (bash tool timeout)

- 새 `crates/jcode-config-types/src/lib.rs::ToolConfig` / `BashToolConfig`:
  - `default_timeout_ms` (default 300_000ms = 5min)
  - `max_timeout_ms` (default 1_200_000ms = 20min)
  - `HARD_CAP_MS = 20min` (절대 상한, config 무시)
  - `effective_default_ms()` / `effective_max_ms()` clamp helper
- `Config::tool: ToolConfig` field 추가
- `src/tool/bash.rs::resolve_timeout_ms(Option<u64>) -> u64`:
  - `requested` 있으면 `clamp(1, max)`
  - 없으면 `default.min(max)`
  - config 읽기는 `crate::config::config()` → M19 hot-reload 자동 적용
- 세 사용처 (foreground/agent-turn/background) 모두 helper 통합
- schema `timeout` description 에 5min default / 20min cap / `[tool.bash]`
  knob 안내 → 모델 (즉, 우리 미래 세션) 이 long-running 시 명시적으로
  `timeout: 600000` 같이 주는 게 권장됨

### 사용자 옵션 (필요시 ~/.jcode/config.toml 에 추가)

```toml
[tool.bash]
default_timeout_ms = 600000   # 10분으로 default 더 늘리기
max_timeout_ms     = 1200000  # 20분 (이미 default)

   ```
