## 요약

Anthropic OAuth path 의 `format_tools(tools, is_oauth=true)` 가 hardcoded 10개 Claude-Code tool 만 광고하고 `tools` 인자를 무시하던 문제를 수정했다. §5.4 권장 형태를 채택해, 실제 allowed `tools` 가 inclusion 을 결정하고 KNOWN OAuth tool 은 schema-only source 로만 사용한다.

## 구현

- `src/provider/anthropic.rs`
  - `oauth_known_tool_schema(name)` 추가.
  - `format_tools(..., is_oauth=true)` 를 tools-driven 로 변경.
  - known alias (`bash` -> `Bash`, `subagent` -> `Agent`, `codesearch` -> `ToolSearch` 등)는 기존 안전 schema 재사용.
  - unknown/jcode-only (`bg`, `swarm`, `memory`, ...) 은 raw name 과 local schema 로 광고.
  - duplicate advertised name 은 first one wins 로 제거.
  - `cache_control` 은 동적 마지막 tool 한 곳에만 부여.
- `src/provider/anthropic_tests.rs`
  - jcode-only tool 광고, cache_control 위치, known duplicate 방지, empty tools, Agent recursion guard, API-key path unchanged 테스트 추가.
  - 기존 M12/M13 schema 테스트는 tools-driven 설계에 맞게 해당 local tool 을 입력하도록 조정.
