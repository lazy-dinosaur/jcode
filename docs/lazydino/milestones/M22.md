# M22 — Subagent Same-Round Spawn Defer

**Priority**: HIGH → Closed
**Status**: ✅ DONE 2026-05-13 — Stage 1 + Stage 2 통합 완료. 1차 by-design 판정은 취소됐고, 실제 fix 적용됨. deploy `lazydino-93e25dae`, fork pushed.

## 한 줄 요약

Main session 한 turn 안에서 여러 tool/subagent 가 emit 됐을 때 turn loop 가
순차 await 하던 문제를 `FuturesUnordered` fan-out 으로 해결. OpenAI
Responses API 의 `parallel_tool_calls` 도 default true 로 활성화되어, provider
emit 가능성 + local dispatch 병렬성 두 layer 모두 보완됨.

## 최종 결론

M22 는 **by-design 이 아니었음**. 증상은 3 layer 의 합성:

1. **Provider emit layer** — OpenAI 는 `parallel_tool_calls` flag 가 필요.
2. **Turn loop dispatch layer** — jcode 가 `for tool_index ... await` 로 순차 실행.
3. **Task tool semantics** — subagent tool 자체는 sub turn 완료까지 await.

Stage 1 로 layer 1, Stage 2 로 layer 2 를 fix. Layer 3 은 의도된 task tool
시맨틱이며, 병렬 dispatch 위에서 정상 동작.
