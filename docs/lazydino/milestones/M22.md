# M22 — Subagent Same-Round Spawn Defer

**Priority**: Medium-High (재현 다시 확인됨, 사용자 워크플로우 직접 차단)
**Status**: 🔴 OPEN, 진단 미수행 (workaround 만으로 운영 중)

## 한 줄 요약

Main session 한 turn 안에서 subagent (`task`/`subagent` tool) 를
두 번 spawn 하면 두 번째가 즉시 실행 안 되고 background 로 deferred.
의도된 직렬화인지, jcode 버그인지 미확정.

## 재현 history (latest first)

- **2026-05-12 라이브** (메인 세션 사용자 보고):
  > "atlas (lazy inventory) → background 갔음 (task_id 503346c9i1)
  > searcher (tsq inventory) → 첫 번째가 background 가는 순간
  > deferred 됨, 못 띄움"
  > "저번 패턴 보니까 두 개 동시에 background 띄우면 두 번째가
  > deferred 되더라."

  → workaround (`Round 당 1 spawn`) 여전히 필요. 사용자가 명시적으로
  "이 버그가 아직도 있네" 라고 보고. 우선순위 격상.

- 2026-05-11 ~ 본 카드 작성 시: workaround 로만 운영.

## Workaround

**Round 당 1 spawn 만.** 현재 모든 세션이 이 규칙 따르고 있음.


### 두 가능성

**(a) Mutex<TurnState> 가 turn 종료까지 잡힘**
- 같은 turn 의 두 번째 spawn 이 lock 못 잡고 queue 로 밀림
- fix: subagent spawn 은 lock free 또는 short-lived lock

**(b) 의도된 single-spawn-per-turn 정책**
- 그렇다면 docs 갱신만 (코드 변경 0)

### 진단 plan

1. `subagent` tool dispatch path 에 trace log 추가 (~20줄)
2. 두 번째 spawn 시점에 state inspect (lock holder, queue 상태)
3. 결과:
   - (a) 라면 fix 시도
   - (b) 라면 limitation 으로 docs 에 명시
