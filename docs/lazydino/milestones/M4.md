# M4 — TUI Interleave 가 Tool 완료 후에야 흡수

**Priority**: Medium (BY-DESIGN, UX 개선 가치)
**Status**: 🟡 BY-DESIGN with caveats

## 한 줄 요약

TUI 에서 사용자가 turn 도는 중 메시지 보내면 "Interleave sent" 표시.
하지만 실제 흡수는 현재 tool 완료 후에야 됨. jcode 원설계가 이렇게 됨.
M5 (Alt+B early race) + M1 (bg delivery) 가 부분적으로 완화됐지만
interleave 자체의 latency 는 여전히 by-design.

## 시작점

### 결정 필요: 진짜 fix 할지 / by-design 그대로 두고 UX 개선만 할지

**옵션 X (진짜 fix)**: tool 도는 중 메시지 즉시 흡수 + AI 가 다음
응답에 반영
- 위험: race condition (tool 결과와 interleave 메시지 순서)
- 작업량: 큼 (turn loop 깊이 건드림)

**옵션 Y (UX 개선만)**: "Interleave sent" 후 실제 흡수까지 시간을
명시적으로 보여줌
- 예: "Interleave sent (queued for after current tool)"
- 예: 진행률 표시 ("tool 3/5 → interleave 흡수 예정")
- 작업량: 작음, 위험 적음

**추천: Y** (M4 entry 도 BY-DESIGN 으로 분류).
