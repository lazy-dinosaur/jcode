### 변경

1. **Swarm safety config 추가**
   - `SwarmConfig { max_active_spawns_per_coordinator, max_active_spawns_per_run }`
   - root config `[swarm]` 으로 노출.
   - project `.jcode/config.toml` / `.jcode/config.local.toml` overlay 지원.
   - env override 우선:
     - `JCODE_MAX_ACTIVE_SPAWNS_PER_COORDINATOR`
     - `JCODE_MAX_ACTIVE_SPAWNS_PER_RUN`
   - default: coordinator=6, run=4. `0` 은 unlimited.

2. **Spawn hard cap**
   - counting rule: `report_back_to_session_id == coordinator` 인 worker 중
     `crashed|closed|disconnected|completed|failed` 가 아닌 모든 상태를 active 로 count.
   - per-coordinator cap 과 per-run cap 둘 다 적용.
   - `handle_comm_spawn` 에서 mutation idempotency 전 pre-check.
   - `spawn_swarm_agent` 진입부에서 공통 re-check.
   - `assign_next prefer_spawn/spawn_if_needed` 도 같은 `spawn_swarm_agent` 경로로 enforce.

3. **CWD allowlist / root pinning**
   - coordinator cwd 를 root 로 사용.
   - requested cwd 와 root 를 `std::fs::canonicalize` 로 symlink resolve 후 `starts_with` 검사.
   - coordinator cwd 가 없으면 backward compat 으로 skip.
   - env-only escape hatch: `JCODE_SWARM_ALLOW_ANY_CWD=1`.
   - reject 메시지에 requested/root 및 override 안내 포함.

4. **Telemetry 확장**
   - spawn tool 응답 suffix 에 `active=<n>/<cap>` 추가.
   - cap=0/unlimited 인 경우 `active=<n>` 만 표시.
   - 기존 `cwd=..., mode=..., run_id=...` 위치와 의미 보존.

5. **Default config docs**
   - `src/config/default_file.rs` 에 `[swarm]` 섹션, env vars, cwd override 문서화.
