## 한 줄 요약

1. **이미지 첨부 silent fail**: wl-paste 가 PNG types 를 감지해서 로그를
   찍지만 (`clipboard_image: wl-paste types: "image/png"`) 실제 byte
   수신/`pending_images.push`/server 전달 중 어딘가에서 조용히 실패.
   사용자에게는 첨부 성공/실패 피드백이 전혀 없음. 디버깅 사이클을
   막음.
2. **leading-space + `/` 가 SlashCommand 모드로 진입**: `" /tmp/..."`
   같이 space 다음 slash 로 시작하면 `composer_mode` 가
   `trim_start().starts_with('/')` 로 SlashCommand 분기 → skill
   suggestion popup 이 떠서 UX 가 혼란. M39 slash-path-prompt-stuck 의
   잔여 정정 요청.
3. **Claude Opus 1m (`claude-opus-4-7[1m]`) 메인 세션에서 안 보임**:
   코드/카탈로그에는 등록되어 있지만 (`anthropic.rs:467`,
   anthropic_tests 통과), `provider/routing.rs:36,46` 의
   `has_extra_usage()` 게이트 때문에 model picker 에 advertised 되지
   않음. subagent route (`variant=max`) 는 `apply_route_variant_to_model`
   로 `[1m]` 변환되어 잘 동작 (metis/planner/sisyphus). 즉 **메인 세션
   model picker 만** 1m 옵션 누락.

## 증거 (live session 에서 수집)

### 증거 1: 이미지 첨부 silent fail

```text
[2026-05-12 23:11:15.149] [INFO] clipboard_image: wl-paste types: "image/png"
[2026-05-12 23:11:32.042] [INFO] clipboard_image: wl-paste types: "image/png"
```

- 두 번의 paste attempt 모두 wl-paste 가 PNG 감지
- 하지만 그 다음 단계 (`wl-paste --type image/png --no-newline` 호출
  → base64 인코딩 → `pending_images.push`) 결과 로그가 전혀 없음
- 사용자 input 박스에는 `[image 1]` placeholder 만 표시되고
  실제 메시지에는 alt-text 만 들어옴 (메인 세션 에이전트가 픽셀을
  못 봄)
- M15 이미지 외부 client 전달과 **다른 문제**: M15 는 attached client
  fanout, M40 은 **첨부 자체가 실패** 하는 단계

### 증거 2: leading-space + `/` slash-mode

- 사용자 정정: "**띄어쓰기 하고 / 했는데 스킬붙잖아**"
- 위치: `src/tui/ui_input.rs:41`
  ```rust
  } else if input.trim_start().starts_with('/') {
      ComposerMode::SlashCommand
  }
  ```
- 같은 sensitivity: `src/tui/app/remote/key_handling.rs:595`,
  `src/tui/app/input.rs:740`, `src/tui/app/input.rs:657` 등
- M39 fix (`parse_invocation` 의 fall-through) 는 send 단계 처리이고,
  UI 모드 진입 자체를 막지 못함

## Drop Candidate

### Candidate A (Image): 진단 로그 + 사용자 피드백 추가 ★ 최소 fix
1. `clipboard_image()` 의 각 단계 (types 감지, --type 호출 status,

   표시
3. 성공 시 `set_status_notice("이미지 첨부됨: {} bytes")` 같은 피드백

### Candidate B (Image): wl-paste --no-newline 빈/실패 결과 처리
- `wl-paste --type image/png --no-newline` 의 status fail 또는 빈
  stdout 케이스가 silent return None 임. 사용자 알림 추가.

### Candidate C (Slash): `composer_mode` 와 `parse_invocation` 일관성
- 옵션 1: `composer_mode` 도 `parse_invocation` 처럼 `!contains(' ')`
  추가 (즉 `" /tmp x"` 는 Chat 모드)
- 옵션 2: leading space 가 있으면 무조건 Chat 모드 (사용자가 일부러
  넣은 escape 시그널로 취급)
- **추천**: 옵션 2 (더 직관적, escape 패턴 활용 가능)

### Candidate D (1m): `has_extra_usage()` 진단 + 게이트 정정
- 1단계: 사용자 계정의 실제 `extra_usage_enabled` 값 확인
  - `jcode usage` 또는 `jcode doctor` 출력에 노출
  - 또는 ad-hoc `cargo run -- usage` 로 fetch 결과 dump
- 2단계: 만약 실제로 user 계정이 1m 지원 안 함 → user-facing 메시지
  ("Opus 1m requires extra usage subscription") picker 에 표시
- 3단계: 만약 user 계정이 1m 지원하는데 fetch 가 못 가져옴 →
  fetch logic bug fix (`usage/anthropic_fetch.rs` 등)
- 4단계: 게이트 정책 검토 — extra_usage_enabled 가 unknown/`false`
  여도 사용자가 명시적으로 picker 에서 골랐다면 attempt 허용
