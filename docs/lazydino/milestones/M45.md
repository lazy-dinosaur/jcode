# M45 — AGENTS.md / instruction discovery parity with opencode

**Priority**: High (instruction-following / monorepo UX)
**Status**: 🔴 OPEN — comparison complete, implementation not started
**Discovered**: 2026-05-13 live user report: “jcode 는 agents.md 를 크게 읽어서 따라가지 않는것같은데?? 구조를 opencode 같은거랑 비교해볼까??”

## 한 줄 요약

jcode 는 현재 project root `AGENTS.md`, `~/.AGENTS.md`, `<project>/.jcode/AGENTS.md`,
그리고 `.jcode/harness/*.md` 를 system prompt 에 정적으로 붙인다. 반면 opencode 는
`AGENTS.md` 외에 `CLAUDE.md`/`CONTEXT.md` fallback, `opencode.json.instructions`
글롭/URL, 그리고 파일 read 시 주변 instruction file 을 추가로 attach 하는 구조가 있다.
monorepo/nested package 에서는 jcode 가 지침을 덜 읽는 체감이 생길 수 있다.

## 현재 jcode 코드 근거

- `src/prompt.rs::build_system_prompt_{full,split}`
  - `load_agents_md_files_from_dir(working_dir)` 결과를 system prompt 에 추가.
- `src/prompt.rs::load_agents_md_files_from_dir_with_config`
  - project: `<project>/AGENTS.md` 만 읽음.
  - global: `~/.AGENTS.md` 만 읽음.
  - private: `<project>/.jcode/AGENTS.md` 만 읽음.
  - project dir 탐색 기준: ancestor 중 `.jcode` 또는 `AGENTS.md` 가 있는 첫 위치.
- `src/prompt.rs::load_prompt_overlay_files_from_dir_with_config`
  - `~/.jcode/prompt-overlay.md`, `<project>/.jcode/harness/*.md`,
    `<project>/.jcode/prompt-overlay.md` 는 지원.
- 지원하지 않는 것:
  - `CLAUDE.md` fallback.
  - `CONTEXT.md` fallback.
  - `~/.config/opencode/AGENTS.md` 호환.
  - `opencode.json`/`jcode config` 의 arbitrary instruction globs/URLs.
  - 파일 read 시 해당 파일 주변의 nested `AGENTS.md` 를 turn 에 추가 주입.

참고: jcode 의 **agent profile markdown** 은 별개로
`.jcode/agents`, `.claude/agents`, `.agents/agents`, `.opencode/agents` 를 읽는다
(`src/agent_profiles_md.rs`). 이것은 subagent/profile 정의이지, general project
instructions (`AGENTS.md`) 로딩과는 다른 경로다.

## opencode 비교 근거

opencode `packages/opencode/src/session/instruction.ts` 기준:

- `FILES = ["AGENTS.md", "CLAUDE.md", "CONTEXT.md"]`
  - Claude compatibility disable flag 가 있으면 `CLAUDE.md` 제외.
- `systemPaths()`:
  - global: `~/.config/opencode/AGENTS.md`, fallback `~/.claude/CLAUDE.md`.
  - project: cwd/worktree 에서 `AGENTS.md` → `CLAUDE.md` → `CONTEXT.md` 순서로 find-up.
  - config `instructions`: local glob, absolute path, `~/...`, URL 지원.
- `system()`:
  - 모든 instruction file/URL content 를 `Instructions from: <path>` 로 system context 에 합침.
- `resolve(messages, filepath, messageID)`:
  - read tool 이 파일을 읽을 때, 그 파일의 directory 에서 project root 까지 올라가며
    nearby instruction file 을 찾아 **한 assistant message 당 한 번** attach.
  - 이미 system instruction 이거나 이미 loaded 된 file 은 dedup.

opencode docs (`packages/web/src/content/docs/rules.mdx`) 도 동일하게 설명:
project/global/Claude fallback/custom `instructions`/external file references.

## 사용자 영향

- repo root 에는 `AGENTS.md` 가 있어도 nested package/subdir 의 `AGENTS.md` 는 무시될 수 있음.
- 기존 Claude Code 프로젝트의 `CLAUDE.md` 를 jcode 가 자동으로 못 읽음.
- opencode 사용자/프로젝트의 `opencode.json.instructions` 기반 rules 를 jcode 가 못 재사용.
- “규칙 파일을 크게 읽고 따라가지 않는다”는 체감이 특히 monorepo, generated docs,
  package-specific conventions 에서 커짐.

## 권장 scope

### Stage 1 — low-risk compatibility

1. Project instruction candidates 를 `AGENTS.md`, `CLAUDE.md`, `CONTEXT.md` 로 확장.
   - same directory 에 여러 개 있으면 opencode 처럼 first match wins.
   - 기존 `AGENTS.md` 동작은 유지.
2. Global fallback 추가:
   - 현 jcode global `~/.AGENTS.md` 유지.
   - optional compatibility: `~/.config/opencode/AGENTS.md`, `~/.claude/CLAUDE.md`.
3. `/info` prompt accounting 에 실제 loaded path/name 표시.

### Stage 2 — custom instruction paths

1. `[prompt] instructions = [...]` config 추가.
   - relative glob, absolute path, `~/...` 지원.
   - URL 은 초기에는 non-goal 또는 5s timeout + cache 로 제한.
2. loaded content 를 `Instructions from: <path>` 형태로 label.
3. tests: glob ordering, missing file tolerant, duplicate path dedup.

### Stage 3 — nested instruction injection

1. Read/grep/edit 등 file-local tool 이 특정 파일을 load 할 때,
   해당 path 주변의 nearest instruction file 을 context 에 주입하는 mechanism 설계.
2. opencode 처럼 message/turn scoped dedup 필요.
3. 위험: 매 tool read 마다 system context 증가 → token budget 관리 필요.
   - first implementation 은 “system reminder once per turn” or “tool result metadata + next turn inject” 중 선택.

## Verification checklist

- Unit tests:
  - project `AGENTS.md` 우선, 없으면 `CLAUDE.md`, 없으면 `CONTEXT.md`.
  - nested cwd 에서 find-up 동작.
  - global fallback ordering.
  - custom `instructions` glob load + deterministic order + dedup.
- Integration/manual:
  - root-only AGENTS 기존 동작 회귀 없음.
  - Claude Code repo (`CLAUDE.md` only) 에서 jcode `/info` 에 loaded 표시.
  - monorepo package-local AGENTS 를 read 후 모델이 해당 규칙을 인지하는지 확인.

## Non-goals / caution

- opencode 의 모든 instruction semantics 를 한 번에 복제하지 말 것.
- remote URL instruction 은 prompt injection/security 면에서 opt-in + timeout + 표시가 필요.
- nested instruction auto-inject 는 token 폭증 위험이 있어 Stage 1/2 이후 별도 검증.
