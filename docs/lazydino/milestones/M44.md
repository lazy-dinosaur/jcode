# M44 — MCP OAuth / authenticated remote MCP support

**Priority**: High (MCP ecosystem/auth compatibility)
**Status**: 🟡 NEXT — confirmed missing in current code; planned for Figma/remote OAuth-required MCP support
**Discovered**: 2026-05-13 live user report: “jcode 에 지금 mcp oauth 가 없네 맞지?”

## 한 줄 요약

현재 jcode 의 MCP 클라이언트는 stdio command 기반 MCP 서버 연결만 지원하고,
remote/authenticated MCP 서버에서 필요한 OAuth/Auth discovery 및 bearer token 흐름이 없다.

## 현재 코드 근거

- `src/mcp/client.rs`
  - `McpClient::connect(name, &McpServerConfig)` 가 child process 를 spawn 해서 stdin/stdout JSON-RPC 로 통신.
  - `Authorization` header 또는 HTTP transport 개념 없음.
- `src/mcp/protocol.rs::McpServerConfig`
  - 필드: `command`, `args`, `env`, `shared`.
  - `url`, `transport`, `headers`, `auth`, `oauth` 같은 remote/auth config 없음.
- `agentgrep oauth src/mcp` 결과: 0 matches.
- `agentgrep Authorization src/mcp` 결과: 0 matches.

즉 provider 쪽 OAuth (`src/auth/oauth.rs`, Anthropic OAuth 등) 는 있지만,
**MCP 서버 자체에 대한 OAuth/Auth** 는 별도 미구현 상태다.

## 사용자 영향

- OAuth 를 요구하는 remote MCP server 를 jcode 에 직접 붙일 수 없음.
- 현재 workaround 는 별도 wrapper command 를 만들어 토큰을 외부에서 주입하거나,
  stdio MCP server 만 사용하는 것.
- Claude Desktop/Claude Code 계열에서 점점 늘어나는 authenticated remote MCP 와 호환성 차이가 생김.

## Live decision 2026-05-13

User wants this before M17/M16 because the immediate goal is connecting Figma and other
remote OAuth-required MCP servers. Current code check confirmed MCP is stdio-only:
`McpClient::connect()` spawns a child process and `McpServerConfig` only has
`command/args/env/shared`. `reqwest` is already available with `json` + `stream`, but
there is no SSE/eventsource helper dependency yet.

Recommended implementation order for practical Figma support:

1. **Stage 1A config/backcompat**: add transport/url/headers/auth config without breaking
   existing `command`-only stdio configs.
2. **Stage 1B static bearer/env-token HTTP transport**: enough for remote MCPs that give
   a token or support manual OAuth outside jcode. This is the fastest Figma unblock if
   Figma exposes a bearer-token MCP endpoint.
3. **Stage 2 streamable HTTP/SSE**: implement MCP remote transport semantics behind the
   same `McpHandle` request interface.
4. **Stage 3 OAuth discovery/login/refresh**: first-class jcode-managed OAuth for servers
   that require dynamic auth.

## 권장 scope

### Stage 1 — transport/config foundation

1. `McpServerConfig` 에 backward-compatible transport fields 추가:
   - `transport`: optional enum, default inferred as `stdio` when `command` is present.
   - `stdio` 기존 default: `command/args/env/shared` 유지.
   - remote values: `http`, `sse`, `streamable_http` (spec/current ecosystem naming 확인 후 최종화).
2. remote endpoint config 추가:
   - `url`
   - optional static `headers`
   - optional `auth` block: `bearer_token_env`, `bearer_token`, future `oauth`.
3. redaction helper 추가: token/header 값은 logs/status/debug snapshots 에 노출 금지.
4. 기존 `~/.jcode/mcp.json` 와 project-local config migration/backcompat 테스트.

### Stage 2 — authenticated HTTP MCP client

1. HTTP/SSE/streamable transport client 구현.
2. JSON-RPC request/response correlation 을 기존 `McpHandle` 과 같은 interface 로 감싸서
   `McpManager`, `McpTool` 쪽 변경을 최소화.
3. bearer token header injection 지원:
   - static token via env reference 우선.
   - inline token 은 지원하더라도 warning + redaction 필수.
   - token redaction in logs/status/debug snapshots.
4. mock HTTP MCP server 로 `initialize` → `tools/list` → `tools/call` end-to-end 테스트.

### Stage 3 — OAuth discovery + login flow

1. MCP Authorization metadata discovery 지원.
2. device code / browser redirect login flow 중 jcode 기존 OAuth UX 와 맞는 방식 채택.
3. token storage/refresh:
   - provider OAuth token store 와 섞지 말고 MCP server id/name scoped storage 권장.
   - refresh 실패 시 user-visible reauth guidance.
4. headless/SSH mode: `--no-browser` 스타일 auth URL 출력 지원.

### Stage 4 — UX/tooling

1. `mcp action="connect"` 또는 별도 CLI 에 OAuth-required 상태 표시.
2. `jcode doctor` 에 MCP remote auth 상태 진단 추가.
3. `README` 의 MCP config examples 갱신.

## Proposed config examples

### Existing stdio remains valid

```json
{
  "servers": {
    "local": {
      "command": "/path/to/server",
      "args": [],
      "env": {},
      "shared": true
    }
  }
}
```

### Remote bearer-token MCP, first unblock target

```json
{
  "servers": {
    "figma": {
      "transport": "streamable_http",
      "url": "https://example.com/mcp",
      "auth": {
        "type": "bearer",
        "token_env": "FIGMA_MCP_TOKEN"
      },
      "shared": true
    }
  }
}
```

## Verification checklist

- Unit tests:
  - `McpServerConfig` 기존 stdio JSON config backward compatible.
  - remote config parse/serialize roundtrip.
  - auth header redaction.
  - token refresh path mock.
- Integration tests:
  - local mock HTTP MCP server with tools/list + tools/call.
  - mock OAuth metadata/device-code/token endpoint.
  - expired token → refresh → retry succeeds.
- Manual/live:
  - OAuth-required remote MCP server 를 등록하고 `mcp` tool 로 목록/호출 확인.
  - headless auth flow 확인.

## Non-goals / caution

- 기존 provider OAuth (Anthropic/Claude 등) 와 MCP OAuth 를 섞지 말 것.
- token 을 `mcp.json` 에 plain text 로 저장하지 말 것. env reference 또는 secure-ish jcode auth storage 사용.
- shared MCP pool 에서 user/session scoped auth 가 섞이지 않도록 server identity + account scope 를 명확히 할 것.
