## 즉시 배제 가설

- **hooks**: `~/.jcode/config.toml` `[hooks] commands = []` (등록 0
  개). `~/.jcode/hooks/test-events.log` 0 byte → fire 안 됨. 무관.
- **M41 wake-up**: log 에 `M41 wake-up` 1 회 발화 (00:00:51.045) —
  bg/inject 가 trigger 한 server-initiated turn (`session_turkey`)
  에서 정상 작동. 메인 세션 (`session_donkey`) hang 과 다른 세션.
  단 부수 가능성 (wake-up 후 stream 이 안 와서 hang) 은 미배제.

## 의심 가설 (다음 세션에서 확인)

1. **websocket 재연결/체크 무한 loop**: `connection_phase=checking websocket`
   이 status bar 에 남아 있는 채로 진행 안 됨. transport layer
   (`websocket/persistent-reuse`) 가 health probe 영역에서 hang.
2. **swarm 다중 세션 간 사이드이펙트**: log 에 `session_donkey`,
   `session_turkey`, `session_moose`, `session_crocodile` 4개 세션
   동시에 active. bg/inject 가 enqueue 됐는데 메인 세션이 그 결과
   기다리며 stream open 못 함?
3. **OpenAI gpt-5.5 transport=auto** 에서 websocket persistent reuse
   가 메시지 1 회 보낸 후 응답 없음. log 의 `idle_age_ms=60038` (60s
   idle) 이 의심.
4. **M22 관련**: subagent 2개 동시 spawn 시 두 번째 deferred. 메인이
   첫 subagent 결과를 기다리며 hang 일 수 있음.
