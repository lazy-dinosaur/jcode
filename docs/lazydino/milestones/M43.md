## 코드 검증 (초기)

- `src/tool/mod.rs:172`: `BgTool` 가 base tool 로 등록.
- `src/tool/task.rs:416-435`: subagent 가 받는 tool set 은
  `self.registry.tool_names()` 전체 minus `{subagent, task, todo*}`
  (recursion 방지). **`bg` 는 명시적으로 안 빠짐** → subagent 에서도
  보여야 한다는 의미.

즉 **두 가지 가능성**:

### 가설 A: 실제로는 subagent 에서 보임, 메인이 추측 오류

- 메인이 "subagent 는 bg 없어" 라고 단정한 게 실측 아닌 추측일 수도.
- 검증: subagent (e.g. `subagent_type=quick`) 의 system prompt /
  tool list 직접 dump 해서 `bg` 포함 여부 확인.

### 가설 B: bg 가 register 되어 있어도 session-scope 때문에 실패

- `task_id=503346c9i1` 는 **메인 세션 (ctx.session_id) 의 task**.
- subagent 는 자기 session 이 따로 (`Session::create` ).
- `background` 모듈이 session-scoped 이면 subagent 에서 같은 task_id
  로 output fetch 시도 → "task not found" 류 에러.
- 그게 "subagent 에 bg 가 없다" 처럼 보였을 수 있음.

## 진단 결과 (1차, 2026-05-13 07:30 KST)

  호출 시도조차 안 일어남).

### 흥미로운 부수 관찰

노출된 tool 이름이 **PascalCase Claude-Code 스타일** (`Bash`, `Read`,
`Edit`, `Grep` ...) — jcode 내부 registry 의 소문자 이름 (`bash`,
`read`, `edit`, `grep`) 과 다름. 즉 anthropic OAuth path 에서
tool advertisement 시 **Claude-Code alias map** 으로 매핑한다는
의미.

`src/tool/mod.rs:313-` 의 `resolve_tool_name` 주석:

> When using OAuth, the API presents tools with Claude Code names
> (e.g. `file_grep`, `shell_exec`).

→ tool 광고 단에서 jcode-only tool (`bg`, `bash_kill`, `bash_output`
등 background 계열) 이 Claude-Code catalog 에 없으니 **광고 자체에서
탈락** 했을 가능성이 가장 큼.

또한 `Agent` (PascalCase) 가 노���됨 — 이건 `subagent`/`task` 의
Claude-Code alias 인데, `task.rs:431` 의 recursion 차단 (`subagent`,
`task` 제거) 이 jcode 내부 이름 기준으로 동작 → alias `Agent` 는
못 잡고 그대로 통과. **recursion 차단도 새는 중**.

### 다음 진단 (swarm worker 위임 예정)

`src/provider/anthropic*.rs` 또는 `src/provider/oauth*.rs` 에서
tool definitions 가 outbound API request 로 직렬화될 때:

1. jcode tool name → Claude-Code alias 매핑하는 함수 위치
2. **매핑 실패 시 어떻게 처리** (drop vs raw name pass-through)
3. `bg`, `bash_output`, `bash_kill` 등 jcode-only tool 들이 모두
   거기서 drop 되는지 (광고 단계) — 또는 raw name 으로는 보내지는데
   server-side 가 unknown tool 로 무시하는지

### Fix 방향 후보

- **A.** OAuth tool advertisement 단에서 unknown jcode tool 은
  raw name 으로 그대로 광고 (Claude API 가 임의 tool name 허용함).
  검증 필요: Claude API 가 alias 안 된 이름 받으면 정상 동작 vs
  4xx 거부.
- **B.** subagent prompt 에 "이 도구들은 메인 세션에서만 호출 가능"
  명시 + main session 가 위임하지 말고 직접 호출하라는 dispatch
  로직.
- **C.** 추가 발견: `Agent` (recursion alias) 도 새는 것 → 같이
  fix. 이건 작은 patch.

Fix 진행은 별도 swarm worker 에 위임 (main session context 보호 정책).
